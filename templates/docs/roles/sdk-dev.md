# SDK 开发角色库

## 角色定义

### SDK 架构师
- 职责：接口设计、模块划分、版本规划
- 输出：API 设计文档、模块结构图、版本策略

### 核心开发
- 职责：核心功能实现、错误处理、兼容性
- 输出：SDK 核心代码、错误码定义、兼容性方案

### 文档工程师
- 职责：API 文档、使用示例、集成指南
- 输出：README、API Reference、Demo 代码

### QA
- 职责：单元测试、集成测试、兼容性测试
- 输出：测试用例、测试报告、覆盖率数据

### PM
- 职责：版本管理、发布计划、文档汇总

## 默认工作流
架构师设计 → 核心开发实现 → QA 测试 → 文档工程师编写 → PM 发布

## 适用场景
- HTTP Client SDK
- 工具库开发
- API Wrapper 封装
- 开源库项目
