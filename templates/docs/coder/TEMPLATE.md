## 质量门禁 (作者自检)

提交前请确认以下 MUST 项全部满足:

### MUST (缺一不可)
- [ ] 改动范围精确到文件路径和函数/方法名
- [ ] 实现步骤有序号、每步有预期产出、有依赖标注
- [ ] 不确定项用 `[?]` 标记，包含"需要谁决策什么"
- [ ] 自测计划覆盖 happy path + ≥2 个 error/edge cases
- [ ] L 级: 方案已通过 Architect 审核 (标注审核人和日期)

### SHOULD (建议满足)
- [ ] 标注了每步预计耗时
- [ ] 引用了架构/交互文档的具体章节

### 代码提交前额外自检
- [ ] 所有改动在方案文档列出的范围内 (无范围蔓延)
- [ ] typecheck / lint 通过
- [ ] 自测计划中全部 case 通过
- [ ] 无 console.log / 调试代码残留
- [ ] 无硬编码密钥/凭据
